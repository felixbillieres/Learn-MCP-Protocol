# Python Exercises for MCP Project 15

Before implementing complex elicitation schemas in MCP, let's practice JSON Schema validation and data structures.

## Exercise 15: Elicitation Schema Validation

This exercise covers:
- JSON Schema creation and validation
- Complex data type definitions
- Required vs optional field handling
- Enum and pattern constraints

### Instructions

1. Open `exercise_15.py`
2. Complete all the TODO comments
3. Run the test: `python3 test_exercise_15.py`

### What you'll practice

- **Schema definitions**: Creating JSON Schema structures
- **Data validation**: Checking data against schemas
- **Type constraints**: Strings, integers, booleans, enums
- **Pattern matching**: Regular expressions for validation

### Key Concepts

- **Required fields**: Must be present in input data
- **Optional fields**: Can be omitted, may have defaults
- **Validation rules**: minLength, maxLength, patterns, enums
- **Error handling**: Meaningful validation error messages

### Expected Output

When all tests pass, you should see:
```
All tests passed! You're ready for MCP Project 15!
```

### Tips

- Use descriptive field names and descriptions
- Test validation with both valid and invalid data
- Handle validation errors gracefully
- Consider edge cases in your schemas

Once you've completed this exercise, you'll know how to create and validate elicitation schemas for MCP tools!
